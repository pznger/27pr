# 第零节课：课前准备与说明

## 零、为什么只有几个学生？

先说点实话。

这个课程目前**正式报名的同学不多**，但这恰恰说明你们走在前面。

### 原因很简单：太早了

这门课面向的是 **2028 年考研 / 保研** 的学生。现在才 2026 年，对绝大多数大二学生来说，「考研」还只是模糊的概念，「科研经历」更是八竿子打不着的事。大多数人的心态是：

> 「才大二，急什么？等大三再说。」

但你们选择了现在开始。

### 为什么现在开始是对的

研究生复试（尤其是保研面试）最看重的是什么？**科研基础**。

- 一篇论文不是一两个月能搞出来的。如果你大四才开始学 Python、学深度学习，导师凭什么要你？
- 反过来，如果你现在把深度学习基础打牢，大二下就能进组干活、大三上学期就能有产出，保研材料上就是实打实的科研经历。
- 大模型这波浪潮至少还能持续五年。你现在投入的时间，**每一分钟都在为未来积累筹码**。

### 后续还有同学可能加入

后面还有几位同学在观望，可能会陆续加入。所以课程节奏会保持稳定，不会因为人少就加速或跳过基础内容。早来的同学可以利用这段时间把基础打得更扎实，后面同学加入时还能帮忙答疑——**教别人是最好的学习方式**。

## 一、正式开课前的准备工作

目标：在第 1 节课直播之前，把下面这些东西全部装好、注册好、跑通验证。

### 1.1 VSCode + Python 环境

> 这是写代码的"写字台"。

**VSCode**（Visual Studio Code）是目前最主流、最轻量的代码编辑器。

1. 下载地址：[https://code.visualstudio.com/](https://code.visualstudio.com/)
2. 安装后建议装的插件：
   - **Python**（Microsoft 官方，必装）
   - **Chinese (Simplified) Language Pack**（中文界面，可选）
   - **Jupyter**（方便看 `.ipynb` 文件）
3. 验证：打开终端（Ctrl+`），输入 `python --version`，能正常输出版本号即可

### 1.2 Typora — Markdown 阅读器

> 课程的所有讲义都是 `.md`（Markdown）格式。你需要一个能正确渲染公式、表格、代码块的阅读器。

**Typora** 是目前最流行的 Markdown 编辑器，所见即所得，支持 LaTeX 数学公式、Mermaid 流程图、表格、代码高亮。

1. 下载地址：[https://typora.io/](https://typora.io/)（免费版功能完全够用）
2. 安装后用 Typora 打开本课程仓库里的任意 `.md` 文件（如 `第一节课/讲义.md`）
3. 确认公式（如 $y = wx + b$）和表格能正常渲染

> **为什么不用记事本看？** 记事本无法渲染 Markdown 格式，公式会显示为 `$$y=wx+b$$` 的原始代码，阅读体验很差。Typora 渲染后的讲义和 PDF 一样清晰。

### 1.3 Anaconda3

> 这是管理 Python 环境的"管家"。不同项目可能依赖不同版本的 Python 或库，Anaconda 可以轻松隔离。

1. 下载地址：[https://www.anaconda.com/download](https://www.anaconda.com/download)
2. 安装参考：[超详细 Anaconda+Python 安装保姆级教程](https://blog.csdn.net/virtuaweaver/article/details/150031173)
3. 安装路径建议放在非 C 盘（如 `D:\software\anaconda3`）
4. 装完后配置**清华镜像源**，下载包时会快很多：

```bash
conda config --add channels https://mirrors.tuna.tsinghua.edu.cn/anaconda/pkgs/free/
conda config --add channels https://mirrors.tuna.tsinghua.edu.cn/anaconda/pkgs/main/
conda config --set show_channel_urls yes
```

5. 验证：打开终端（Anaconda Prompt 或 VSCode 终端），输入 `conda --version`，正常输出即成功

### 1.4 网络环境：解决梯子问题

> 做深度学习开发，**网络是最大的隐形门槛**。很多关键资源都在国外服务器上。

**哪些场景需要梯子（科学上网）？**

| 场景 | 说明 |
| --- | --- |
| GitHub | 克隆代码仓库、下载 Release 文件。不开梯子经常 10KB/s 甚至直接超时 |
| HuggingFace | 下载预训练模型（CLIP、BERT 等）。不开梯子大概率失败或极慢 |
| PyPI / Conda | 部分包（如 `torch`）默认从海外源下载，不开梯子只有几十 KB/s |
| Google Scholar / arXiv | 查论文、看最新研究 |

**解决方案（按推荐顺序）**：

1. **自备梯子（推荐）**：找一个稳定的机场 / VPN 服务，月费通常 10-30 元。这是最省心、一劳永逸的方案。建议向身边做技术的同学打听口碑好的服务商。

2. **清华镜像源（部分替代）**：
   - PyPI 包：`pip install xxx -i https://pypi.tuna.tsinghua.edu.cn/simple`
   - Conda 包：前面已配置清华镜像源
   - HuggingFace 镜像：`export HF_ENDPOINT=https://hf-mirror.com`（在终端执行一次即可）

   > ⚠️ 镜像源可以解决大部分包下载问题，但 GitHub clone 和部分小众模型仍需梯子。强烈建议准备梯子。

3. **Steam++ (Watt Toolkit)**：免费工具，可以加速 GitHub 访问，但不够稳定，仅作备选。

> **一句话总结**：没有梯子做深度学习等于瘸了一条腿。10 块钱一个月的节点，省的是你无数次对着 "Connection timed out" 干瞪眼的时间。

### 1.5 硅基流动（SiliconFlow）注册账号

> 这是调用大模型 API 的平台。课程中 Tiny-RAG 和 Tiny-Agent 的 Embedding、LLM 推理都要走这里。

1. 注册地址：[https://siliconflow.cn/](https://siliconflow.cn/)
2. 注册 → 登录 → 进入控制台，找到 **API 密钥**（API Key）
3. 把 API Key 复制并保存好（后面会频繁用到）
4. **新用户通常有免费额度**，课程前半段基本够用；如果用完了也可以小额充值（很便宜，几块钱能用很久）

**为什么用硅基流动而不是直接用 OpenAI？**

- OpenAI 需要海外手机号 + 外币信用卡，门槛高
- 硅基流动是国内平台，手机号直接注册，支持微信/支付宝充值
- 接口格式完全兼容 OpenAI SDK，代码写法一模一样
- 提供免费的 Embedding 模型（BGE-M3）和多种国产大模型

### 1.6 AutoDL 注册账号

> 这是租 GPU 云服务器的平台。本地电脑没显卡？没关系，AutoDL 上几毛钱一小时就能租到 A100。

1. 注册地址：[https://www.autodl.com/](https://www.autodl.com/)
2. 注册 → 登录 → 进入控制台
3. **先不急着租机器**，课程前期用 CPU 就能跑大部分内容
4. 等到需要 GPU 的时候（比如第三节课训练模块、第四节课跑大模型推理），老师会教大家怎么选配置、怎么用

**AutoDL 的优势**：

- 按小时计费，不用就关机，不浪费钱
- 预装了 PyTorch、CUDA 等常用环境，开箱即用
- 新人注册有赠送金额，基本够课程使用

### 1.7 Cursor / Claude Code（二选一）

> 这是 AI 编程助手。不是"帮你写作业"的作弊器，而是你的 **24 小时助教**——读代码、解释概念、debug、生成实验脚本。

#### Cursor（推荐，图形界面）

1. 下载地址：[https://cursor.com/](https://cursor.com/)
2. 安装参考：[Cursor 下载安装以及使用教程](https://blog.csdn.net/weixin_36467887/article/details/159619159)
3. 免费版每月有 2000 次 AI 对话，课程够用
4. 核心功能：
   - **Chat（Ctrl+L）**：选中一段代码，让 AI 解释它在做什么
   - **Composer（Ctrl+I）**：让 AI 帮你写/改代码
   - **Tab 补全**：自动续写代码，大幅提升打字效率

#### Claude Code（备选，终端命令行）

1. 如果你更喜欢在终端里工作（不用鼠标）、或者 Cursor 的免费额度用完了

```bash
npm install -g @anthropic-ai/claude-code
claude
```

3. 同样需要 Anthropic 的 API Key（跟硅基流动一样，注册即可）

> **选哪个？** 零基础 → Cursor（有中文界面，图形操作直观）；有编程基础 → 都可以，Claude Code 在终端里更灵活。

## 三、心态准备

### 这不是一门"听完就会"的课

深度学习是**实践学科**。你不需要听完一节课就全懂——第一遍有 30% 的印象就很好了。真正的理解来自：

1. 看代码 → 跑代码 → 改参数 → 观察结果变化 → 看懂了
2. 这个过程在每次自习任务里重复

### 善用 AI

- 不懂的概念 → 直接问 Cursor / Claude Code：「ReLU 是什么？用小学生能听懂的话解释」
- 看不懂的代码 → 选中 → 「解释这段代码每一行在做什么」
- 报错了 → 把报错信息复制给 AI → 它会告诉你原因和修法

**但注意**：AI 是工具，不是替代品。最终理解要靠你自己反复跑代码、对比实验结果。AI 帮你省掉的是"卡在环境配置上三天"的痛苦，而不是"理解梯度下降"的思考过程。

### 坚持八周，你会不一样

八周后，你将能够：
- 独立阅读并运行深度学习项目代码
- 理解大模型（LLM）的基本原理和应用方式
- 动手搭建自己的 RAG 知识库和 AI Agent
- 看懂 CLIP、Transformer 等经典论文的核心思路

**这些能力，足够支撑你在大三进任何实验室或者应对985考研复试。**
